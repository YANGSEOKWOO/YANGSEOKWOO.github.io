## 왜 블로그를 만들었나

배운 것을 기록하고 정리하는 습관을 만들고 싶었다. 노션에 정리하던 것을 공개 블로그로 옮기기로 했다.

## 기술 스택 선택

- **Jekyll** — GitHub Pages 네이티브 지원, 마크다운으로 글 작성
- **GitHub Pages** — 무료 호스팅, push만 하면 자동 배포

## 배운 점

1. Jekyll의 레이아웃 시스템 (`_layouts`, `_includes`)
2. Front matter를 활용한 카테고리 분류
3. CSS Grid로 카드형 레이아웃 구성

## 다음 할 일

- 글을 꾸준히 쓰는 것이 가장 중요하다
- 댓글 기능 추가 고려 (giscus)
